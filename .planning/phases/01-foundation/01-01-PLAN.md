## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer machine -> npm registry | Package install (Next.js scaffold + clsx + tailwind-merge) downloads code that runs at install/build time |
| developer machine -> Google fonts | next/font fetches font binaries at build time; bakes them into the bundle |
| developer disk <-> repo working tree | `.env.local` may exist on disk during dev; must never reach git |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | T (Tampering) | npm dependencies installed via create-next-app | mitigate | Commit `package-lock.json` to lock exact versions; only first-party Next.js / Tailwind / shadcn deps allowed (no transitive surprises added in this plan) |
| T-01-02 | I (Information disclosure) | `.env.local` accidentally committed | mitigate | `.gitignore` already covers `.env*.local`; Task 1 verifies the rule is present; only `.env.local.example` (placeholders, no secrets) is committed |
| T-01-03 | I (Information disclosure) | API keys leaked into client bundle via misuse of process.env | mitigate | No `process.env.*_KEY` reads exist in this plan's code; Plan 02 introduces `lib/env.ts` as the single chokepoint and forbids `process.env` elsewhere (D-21) |
| T-01-04 | S (Spoofing) | next/font fetches Google fonts at build time | accept | Build-time fetch is bundled into the app; no runtime fetch from Google; risk is npm-side only and equivalent to T-01-01 |
| T-01-05 | T (Tampering) | Editing existing `.gitignore` could remove rules and leak future secrets | mitigate | Per CLAUDE.md global §4.11, use Edit tool to merge — never replace; Task 1 verifies presence of `.env*.local`, `node_modules/`, `.next/` rules after edit |
