<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer machine -> GitHub | First push contains the entire repo history; any committed secret leaks publicly if repo is public |
| GitHub repo -> Vercel build | Vercel pulls source on each push; build env contains the encrypted env vars |
| Vercel runtime -> public Internet | The deploy URL is unauthenticated; anyone with the URL can hit any route, including /api/health |
| Browser -> Vercel deployment | DESIGN-04 requires reachable interactive elements over standard HTTPS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-12 | I (Information disclosure) | API keys committed to public GitHub repo | mitigate | `.env.local` is in `.gitignore` (Plan 01) and Task 2 Step 1 verifies it is not tracked; Task 2 Step 6 greps git history for key values; Vercel env vars are encrypted-at-rest in dashboard, not in repo |
| T-01-13 | I (Information disclosure) | The public-trial Tavily token `TVLY-HF9ETJRW` is committed in `.planning/PROJECT.md` line 59 | accept | Documented in PROJECT.md as a free-credit trial token shared with the hackathon; user accepted the disclosure at project init; the production Tavily key is set via Vercel env, not from this token |
| T-01-14 | D (Denial of service) | Public URL hit-rate exceeding free tier | accept | Vercel free tier has generous limits; hackathon traffic is a few judges over 24h; if exceeded, Vercel queues and degrades, no security impact |
| T-01-15 | I (Information disclosure) | `/api/health` reveals which API keys are configured to anonymous callers | accept | The endpoint returns booleans only, not values; revealing "this app uses Gemini + Tavily" is acceptable for a hackathon demo (the stack is already public in the project README and PROJECT.md) |
| T-01-16 | T (Tampering) | Anyone with push access to GitHub can deploy to Vercel | mitigate | Single-user repo for hackathon; no external collaborators; Vercel deploy gating is GitHub auth on the dev's account; risk equals dev's GitHub account security |
| T-01-17 | S (Spoofing) | Vercel preview-URL link forwarded to a third party who could phish using the official-looking URL | accept | All Vercel URLs end in `.vercel.app` — phishing risk is generic to the platform, not specific to this app; out of scope for a 24h hackathon |
| T-01-18 | I (Information disclosure) | Vercel build logs containing env-var values | mitigate | Vercel automatically masks env-var values in build logs by default; no special action needed |
</threat_model>
