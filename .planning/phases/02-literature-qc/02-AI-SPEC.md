## 1. System Classification

**System Type:** **Hybrid — Search-Grounded Structured Extraction.** A single-turn pipeline that (1) retrieves evidence from external search (Tavily across arXiv, Semantic Scholar, protocols.io), and (2) calls an LLM to score the evidence into a typed verdict object with citations selected from the retrieved set. Closer to RAG than to a multi-agent system: there is no agent loop, no tool-calling, and no autonomous planning — just retrieve → judge → render.

**Description:**
Sextant's Phase 2 lit-QC pipeline takes a single scientist-authored hypothesis (~1-3 sentences), runs one Tavily search, and asks Gemini Flash-Lite to read the top-10 results and emit a discriminated-union response: a novelty verdict (`not-found` / `similar-work-exists` / `exact-match-found`) with exactly 3 citations drawn from the search results, OR a clarifying question if the hypothesis is ambiguous, OR a typed `no-evidence` / `error` state. "Good" means: the verdict is conservative, every citation URL was actually returned by Tavily (no hallucination), the response streams in <8 seconds, and a working scientist would trust the verdict enough to refine their hypothesis or proceed.

**Critical Failure Modes:**
1. **Citation confabulation** — Gemini emits a citation whose URL was not in the Tavily input. Violates CLAUDE.md hard rule #1; collapses judge trust. Mitigated by post-stream URL-provenance validation (D-37).
2. **Over-confident `not-found` verdict** — Gemini says "novel" when Tavily simply returned weak results. Misleads the scientist into running a duplicated experiment. Mitigated by conservative-thresholds prompt (D-35) and `no-evidence` fallback state.
3. **Clarification spam** — Model asks for clarification on every freeform input, breaking the at-most-one-clarification rule (INPUT-02). Mitigated by explicit prompt threshold + client-side enforcement.
4. **Latency cliff** — End-to-end exceeds 10s, breaking LITQC roadmap criterion. Mitigated by streamObject (perceived latency) + <8s internal budget + fallback to gemini-2.5-flash-lite.
5. **Prompt injection via hypothesis text** — A hostile hypothesis attempts to override the verdict label or exfiltrate the system prompt. Mitigated by treating user input as data only (no instruction-following), Zod-enum-constrained verdict field, and refusing any output not matching the schema.

## 2. Framework Decision

**Selected Framework:** **Vercel AI SDK** (`ai` package) + `@ai-sdk/google` provider for Gemini, in a Next.js 16 App Router server route. Tavily for grounding via direct REST (no SDK abstraction).

**Version:**
- `ai`: ^4.x (latest stable on Vercel AI SDK 4 — supports `streamObject` with Zod, `useObject` client hook, AI SDK UI streaming protocol)
- `@ai-sdk/google`: ^1.x (current Google provider with Gemini 3.x preview support, single env key `GOOGLE_GENERATIVE_AI_API_KEY`)
- `zod`: ^3.x (already in project from `src/lib/env.ts`)
- Tavily: REST API at `https://api.tavily.com/search`, no SDK installed (one HTTP call, not worth a dependency)

**Rationale:**
- **System type fit:** Search-grounded structured extraction is exactly what `streamObject` + Zod schema is built for. The SDK abstracts the model + provider behind a uniform surface so the same code works against Gemini, Claude, or OpenAI by changing one model-ID string.
- **Stack fit:** Project is single-language TypeScript on Next.js 16 (locked in CLAUDE.md hard rule #4). LangChain.js / LlamaIndex.ts add weight for capabilities Phase 2 doesn't need (chains, retrievers, vector stores). CrewAI / LangGraph are agent-loop frameworks; Phase 2 has no agent loop.
- **Streaming UX:** `streamObject` + `useObject` deliver field-by-field streaming of the verdict object with zero glue code. That hits the demo-feel goal in CONTEXT.md D-38.
- **Hackathon stage:** Solo dev, 24h window, hosted on Vercel free tier. Vercel-native tooling minimizes deploy friction (env vars, edge runtime if needed, AI SDK observability hooks).
- **Provider lock window:** Anthropic credits expired at hackathon start; Gemini 3.x family is credit-positive in early 2026 and competitive on structured-JSON output. SDK abstracts so the swap to OpenAI / Anthropic is one model-ID change, not a rewrite.

**Alternatives Considered:**

| Framework | Ruled Out Because |
|-----------|------------------|
| **LangChain.js** | Heavyweight for one retrieve-and-judge call; adds prompt-template, output-parser, and retriever abstractions we'd never use again in Phase 2. Would re-introduce dependency churn during the 24h build. |
| **LlamaIndex.ts** | Optimized for index-and-retrieve over local corpora. Tavily already returns ranked results; we don't need an indexing layer. |
| **CrewAI / LangGraph** | Multi-agent / state-machine frameworks. Phase 2 is a single linear pipeline; agent orchestration arrives in Phase 3 with its own design (4 agents over Vercel AI SDK `streamText` + `tools`). |
| **OpenAI Agents SDK** | OpenAI-locked; we're on Gemini for Phase 2. Could revisit in Phase 3 if Gemini hits rate limits. |
| **Direct `fetch` to Gemini REST** | Loses streaming structured-output ergonomics, retry logic, schema validation. The SDK is a thin layer over the same REST surface and saves ~50 lines of boilerplate. |

**Vendor Lock-In Accepted:** **Partial.**
- **Provider lock (low):** Vercel AI SDK abstracts Google / OpenAI / Anthropic. Switching providers is a model-ID change + one env key. Acceptable.
- **Hosting lock (medium):** Vercel-native AI SDK + Next.js App Router. Hackathon hosting on Vercel free tier is locked anyway. Acceptable for the demo window; portable to any Node host post-hackathon if needed.
- **Tavily lock (low):** Tavily is the only literature-search API in the design. Replacing it (e.g., with Exa, Perplexity API) would be ~30 LoC in `src/lib/tavily.ts`. Acceptable.
